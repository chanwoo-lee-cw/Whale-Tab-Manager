# 사이드바 앱 (Sidebar) 기능 요구사항

## 목적
Whale 브라우저의 사이드바에서 저장된 탭 그룹을 바로 확인하고 이동할 수 있도록
랜딩 페이지를 사이드바 앱으로 제공한다.

## 기능 요구사항

| ID | 요구사항 |
|----|---------|
| SB-01 | 사이드바에서 저장된 탭 그룹 목록을 확인할 수 있다 |
| SB-02 | 사이드바에서 탭 항목 클릭 시 해당 URL로 이동할 수 있다 |
| SB-03 | 사이드바에서 그룹 내 모든 탭을 한 번에 열 수 있다 |
| SB-04 | 사이드바에서 탭/그룹을 삭제할 수 있다 |
| SB-05 | 사이드바에서 탭을 검색할 수 있다 |
| SB-06 | 사이드바 UI는 좁은 너비(320px 이하)에 최적화된다 |

## 비기능 요구사항
- Whale 전용 `whale.sidebarAction` API를 사용해 사이드바에 등록한다
- 사이드바 페이지는 랜딩 페이지(`landing.html`)와 동일한 JS 로직을 재사용한다
- 사이드바와 랜딩 페이지는 동일한 storage를 공유하므로 한쪽 변경이 실시간으로 반영된다

## 범위 밖 (Out of Scope)
- 사이드바에서 탭 저장 기능 (저장은 팝업에서만 수행)
- 사이드바에서 그룹 병합
- 일반 Chrome에서의 사이드바 동작 (`whale.sidebarAction`은 Whale 전용)

## 수용 기준 (Acceptance Criteria)
- Whale 브라우저 사이드바에서 확장 앱 아이콘 클릭 시 탭 그룹 목록이 표시된다
- 사이드바에서 탭 클릭 시 해당 URL이 열린다
- 팝업에서 탭을 저장하면 사이드바 목록이 자동으로 갱신된다
- 320px 너비에서 레이아웃이 깨지지 않는다
