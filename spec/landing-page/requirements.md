# 랜딩 페이지 (Landing Page) 기능 요구사항

## 목적
저장된 탭 그룹을 한눈에 확인하고, 저장된 탭으로 이동하거나 그룹/탭을 관리할 수 있는 전용 페이지를 제공한다.

## 기능 요구사항

| ID | 요구사항 |
|----|---------|
| LP-01 | 저장된 모든 탭 그룹을 목록으로 표시한다 |
| LP-02 | 각 그룹은 그룹 이름, 저장 시각, 탭 수를 표시한다 |
| LP-03 | 각 탭 항목은 favicon, title, url을 표시한다 |
| LP-04 | 탭 항목 클릭 시 해당 URL을 새 탭 또는 현재 탭으로 열 수 있다 |
| LP-05 | 그룹 내 모든 탭을 한 번에 열 수 있다 |
| LP-06 | 그룹 이름을 랜딩 페이지에서 직접 변경할 수 있다 |
| LP-07 | 그룹 전체를 삭제할 수 있다 |
| LP-08 | 그룹 내 특정 탭을 삭제할 수 있다 |
| LP-09 | 저장된 그룹이 없을 경우 빈 상태 안내 메시지를 표시한다 |
| LP-10 | 검색어 입력 시 그룹 이름·탭 title·URL을 기준으로 실시간 필터링한다 |
| LP-11 | 두 개 이상의 그룹을 선택해 하나의 그룹으로 병합할 수 있다 |
| LP-12 | 병합된 그룹의 이름은 사용자가 지정하거나 기본값(첫 번째 그룹 이름)으로 설정된다 |
| LP-13 | 각 그룹 카드는 탭 목록을 접거나 펼칠 수 있다 (Fold/Unfold) |
| LP-14 | 헤더 영역에 모든 그룹을 한 번에 접거나 펼치는 버튼을 제공한다 |
| LP-15 | 사용자가 빈 탭 세션(이름만 있고 탭은 없는 그룹)을 직접 추가할 수 있다 |
| LP-16 | 콘텐츠 좌측에 세션 이름 목록 사이드바를 표시하여 클릭 시 해당 그룹으로 빠르게 스크롤한다 |

## 비기능 요구사항
- 랜딩 페이지는 확장 프로그램의 옵션 페이지 또는 별도 HTML 페이지로 제공된다
- 페이지 로드 시 storage에서 최신 데이터를 읽어 렌더링한다
- storage 변경 이벤트(`chrome.storage.onChanged`)를 구독하여 실시간 반영한다
- 검색은 클라이언트 사이드 필터링으로 처리한다 (storage 재조회 없음)
- UI 디자인은 frontend-design 플러그인을 기준으로 구현한다

## 범위 밖 (Out of Scope)
- 탭 간 드래그 앤 드롭 이동
- 탭 메모/태그 기능
- 병합 시 중복 URL 자동 제거

## 수용 기준 (Acceptance Criteria)
- 랜딩 페이지 진입 시 저장된 모든 그룹이 최신순으로 표시된다
- 탭 클릭 시 해당 URL이 새 탭으로 열린다
- "그룹 전체 열기" 클릭 시 해당 그룹의 모든 탭이 열린다
- 그룹/탭 삭제 후 페이지가 즉시 업데이트된다
- 저장된 그룹이 없으면 "저장된 탭이 없습니다" 메시지가 표시된다
- 검색어 입력 시 일치하는 그룹/탭만 표시되고, 검색어 초기화 시 전체가 다시 표시된다
- 두 그룹 선택 후 병합 시 하나의 그룹으로 합쳐지고 원래 그룹은 삭제된다
- 그룹 카드의 Fold 버튼 클릭 시 탭 목록이 접히고, 다시 클릭 시 펼쳐진다
- "모두 접기" 클릭 시 모든 그룹의 탭 목록이 접히고, "모두 펼치기" 클릭 시 전체가 펼쳐진다
- "새 세션 추가" 클릭 시 이름 입력 후 빈 그룹이 생성되어 목록에 표시된다
- 사이드바의 세션 이름 클릭 시 해당 그룹 카드로 부드럽게 스크롤된다
- 사이드바는 현재 뷰포트에 보이는 그룹을 활성 항목으로 표시한다 (스크롤 연동)
